description: 引導撰寫高品質的 Spec / User Stories
argument-hint: [專案名稱或需求描述]
---

使用 spec-writing skill 引導使用者撰寫高品質的 Spec。

遵循三階段流程：

**Phase 1: 釐清目標**
1. 詢問使用者：這個需求的受益者是誰？（終端使用者、開發團隊、管理者）
2. 請使用者用一句話描述：「完成這個 Story 後，世界有什麼不同？」
3. 請使用者列出「不做什麼」（No-gos）

**Phase 2: 撰寫 Stories**
1. 協助判斷每項需求是 User Story 還是 Enabler Story
2. 依據類型撰寫 Story 本體
3. 撰寫 Outcome-Focused 的驗收標準
4. 為 Enabler Story 撰寫價值陳述
5. 標記優先序（P0/P1/P2/P3）

**Phase 3: 品質檢查**
完成撰寫後，使用 spec-reviewer agent 進行品質檢查：
- INVEST 逐條檢查
- 反模式掃描

如果使用者提供了專案名稱或需求描述（$ARGUMENTS），先分析需求再開始流程。
如果沒有提供，直接詢問使用者想要撰寫什麼需求的 spec。

全程使用中文與使用者互動。
